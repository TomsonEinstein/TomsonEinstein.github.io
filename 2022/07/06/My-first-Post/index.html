当心哪些问题

  • _config.yml
    • 更改URL(你要发布的站点)
    • 更改说明部分，让别人知道这是谁，这是什么
    • 更改主题，不要用原生，推荐NeXT
    • 写好search
    • **deploy:**一定要配置好SSH和git
  • 如果遇到了问题
    • GitHub pages 用的是 Jekyll
    • 直接push本地上去是不行的，需要hexo d
    • hexo clean ; hexo g ; hexo s ;hexo d 一同操作重启试试
    • 如果明显有网页但是有问题，建议F12自己看看哪里出了问题而不是到处乱查
    • GitHub pages有可能慢，可以用Gitee代替
  • 你要掌握什么
    • 熟练使用Git
    • 熟练使用markdown
    • Node.js有使用背景
    • 有一颗钻研的心和自己思考的脑子，而不是只会fork
